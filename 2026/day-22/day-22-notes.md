# Day 22 – Git Understanding

## git add vs git commit

- git add moves changes to the staging area.
- git commit permanently records staged changes in repository history.

## Staging Area

- The staging area allows selective commits.
- It prevents committing unintended changes.

## git log

Shows:
- Commit hash
- Author
- Date
- Commit message

## .git Folder

- Contains all repository metadata and history.
- If deleted, version control history is lost.

## Working Directory vs Staging vs Repository

Working Directory:
- Where files are edited.

Staging Area:
- Where changes are prepared.

Repository:
- Where commits are permanently stored.
